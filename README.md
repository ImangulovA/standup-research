# Stand-up Research — словарный запас русских комиков

Корпусное исследование лексического разнообразия русскоязычных стендап-комиков,
по аналогии с исследованием рэперов Ильи Иноземцева
(«У кого из рэперов больше словарный запас?»).

## Идея

Мы качаем полные стендап-концерты с YouTube, извлекаем транскрипт (субтитры),
очищаем его, лемматизируем и считаем, сколько **уникальных лемм** встречается
в **первых 25 000 слов** очищенного корпуса комика. Метрика повторяет методику
Иноземцева, поэтому корпуса совместимы, и его материалы по комикам можно
класть в тот же корпус.

## Методология (совместима с Иноземцевым)

- **Токенизация + лемматизация:** `razdel` + `pymorphy3`.
- **Очистка:** удаляем разметку субтитров и **точные повторы строк**.
- **Метрика:** число уникальных лемм в первых **25 000 слов** очищенного
  корпуса артиста (все концерты комика склеиваются). Корпус < 25 000 слов —
  «бледный круг», пониженная надёжность.
- На концерт храним только **текст** и **леммы** (плюс лёгкая мета).

## Структура

```
standup_research/
  corpus/<комик>/<концерт>/
      transcript.txt   очищенный текст
      lemmas.txt       поток лемм (только слова), для метрики 25k
      tokens.tsv       token<TAB>lemma
      meta.json        url, комик, название, год, длительность, счётчики
  inozemtsev/          материалы Иноземцева по комикам (в том же формате)
  scripts/             fetch/clean/lemmatize/report
  sources_smeshno.json распарсенный каталог концертов
  report.json          агрегат по артистам (после report.py)
```

## Команды

```bash
# добавить один концерт
./.venv/bin/python scripts/process.py "<youtube_url>" --artist "Имя" [--year 2023]

# пересчитать рейтинг по всем комикам
./.venv/bin/python scripts/report.py
```

## Что НЕ публикуется (важно)

Дословные транскрипты (`corpus/`), сырые субтитры (`raw_subs/`) и локальная
копия чужого сайта-референса (`reference/`) **не выкладываются публично** и
исключены через `.gitignore`. Это чужой авторский материал (тексты выступлений
комиков), и мы **не показываем его людям**: в репозитории только наш код, доки и
агрегированная статистика (`report.json`, `sources_smeshno.json`). Транскрипты
существуют только локально как промежуточные данные для подсчёта метрики.
