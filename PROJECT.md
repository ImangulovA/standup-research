# Stand-up Research — проектный файл

Корпусное исследование словарного запаса русских стендап-комиков, по аналогии с
исследованием рэперов Ильи Иноземцева. Публикуем на каналах **Невежда** и
**Шаппелл позвонит**.

## Бренды и каналы
- **Невежда** — Telegram: https://t.me/newezha (nerdy/zadrotsky юмор, комедия, рэп)
- **Шаппелл позвонит** — Telegram: https://t.me/Chappellepozvonit (хайповая
  ассистентка-комикесса-инсайдерка; голос/персона проекта)
- Инфографику делаем под со-брендинг этих двух каналов.

## Цель
Скачать стендап-концерты с YouTube → транскрипт (субтитры) → очистка →
лемматизация → метрика «уникальные леммы в первых 25 000 слов» на комика.
Итог: интерактивная инфографика «у кого из комиков богаче лексика», как у
Иноземцева, но про стендап и с нашим брендингом.

## Методология (1:1 с Иноземцевым, + наши правки)
- Токенизация `razdel` + лемматизация `pymorphy3`.
- Очистка субтитров, удаление точных повторов строк.
- Метрика: уникальные леммы в первых **25 000 слов** объединённого корпуса
  комика. Корпус < 25 000 слов = «бледный круг» (ниже надёжность).
- **Только русский (наша правка):** считаем **только кириллические** токены,
  латиница/английский выбрасываются. Переводные и англоязычные треки
  субтитров не используем — трек выбирается по реальной доле кириллицы
  (`_pick_vtt`), а не по коду языка (ловушка: у некоторых видео под кодом `ru`
  залит английский перевод — напр. Соболев «Стендап не на ТНТ»).

## Пайплайн (`scripts/`)
- `process.py` — один концерт: скачать субтитры → очистить → лемматизировать →
  сложить в `corpus/`. Кэширует сырые `.vtt` в `raw_subs/` навсегда, дедуп по
  `video_id`, офлайн-пропуск готовых по id из URL.
- `batch.py` — шардовый прогон (`--shard i --nshards K`).
- `clean_vtt.py`, `lemmatize.py` (кириллица-only).
- `normalize.py` — перевыбор оригинального ru-трека + перелемматизация всего
  корпуса + аудит на английский.
- `report.py` — рейтинг по комикам → `report.json`.

Один концерт: `./.venv/bin/python scripts/process.py "<url>" --artist "Имя" [--year 2023]`

## Хранение
`corpus/<комик>/<название>_<vid>/`: `transcript.txt`, `lemmas.txt`,
`tokens.tsv`, `meta.json` (+ `cyr_ratio`, `sub_track`). Сырые субтитры —
`raw_subs/`. Референс сайта Иноземцева — `reference/`.

## Что НЕ показываем / не публикуем
Дословные транскрипты (`corpus/`), сырые субтитры (`raw_subs/`) и копия чужого
сайта-референса (`reference/`) — **чужой авторский материал**. Мы **не
показываем эти вещи людям** и не выкладываем их: всё это в `.gitignore`,
существует только локально как промежуточные данные для подсчёта метрики.
Публично — только код, доки и агрегаты (`report.json`, `sources_smeshno.json`).

## Детектор «монолог vs разговор» (2026-07-09)
Многие ролики в поиске/на каналах — не сольники, а подкасты, интервью, ЧБД,
прожарки, ток-шоу, импровизации, кроуд-хэви панели. Их субтитры содержат чужую
речь и портят метрику (уникальные леммы одного комика). Отсекаем так:
- **Главный сигнал:** авто-субтитры YouTube ставят маркер `>>` (в файле
  `&gt;&gt;`) при смене говорящего. Монолог — почти без них, разговор — много.
  Эмпирика по 645 заведомо-концертным трекам: медиана **0/мин**, p95 1.7, p99
  4.7, `<1%` выше 5/мин. Разговорный контент — **30–50/мин**. Зазор огромный.
  Важно: `>>` считает именно чужую речь, реально попавшую в транскрипт (чистый
  кроуд-ворк без микрофона у зрителя маркеров не даёт — и правильно, там только
  слова комика).
- **Пороги (маркеров/мин):** ≤5 = концерт (брать); 5–15 = ревью (редкий
  кроуд-хэви концерт); >15 = разговор (выкинуть).
- **Дешёвые пре-фильтры (без скачивания):** длительность ≥ **10 мин** (нижняя
  граница; 25 мин НЕ работает — режет короткие сольники); хард-блок по заголовку
  (подкаст/интервью/ЧБД/прожарка/ток-шоу/импровизация/…); коллаб «A x B / vs /
  feat» — мягкий сигнал (→ review), т.к. одиночная « х » иногда ложно срабатывает.
- Не путать с кириллическими предлогами: «Концерт **в** Киеве», «Тип **с**
  Ростова» — это НЕ коллаб (старый регекс их ловил, пофикшено).
- Реализация: `scripts/classify.py` (`classify()`, `classify_video_id()`).
  Аудит текущего корпуса: 318 keep / 0 reject / 3 review (все три — легитимные
  кроуд-хэви концерты). Корпус чист.

## Пайплайн расширения (discover → review → ingest)
1. `scripts/discover.py --targets [--artist "Имя"] [--fetch]` — ytsearch по
   комикам (по умолчанию — близкие к порогу, 18k–25k слов), дешёвый пре-фильтр,
   дедуп по `video_id` с корпусом; `--fetch` качает субтитры и ставит вердикт.
   Пишет `expansion/candidates.json`.
2. `scripts/build_review.py` → `expansion/review.html` — self-contained страница
   ревью: кандидаты по комикам с вердиктом/маркерами/длительностью/ссылкой,
   кнопки принять/отклонить (дефолт = вердикт детектора), localStorage,
   «Экспорт принятых» → JSON. Открывать двойным кликом (из песочницы `open`
   даёт error -54).
3. Сохранить экспорт в `expansion/approved.json` → `scripts/ingest.py` заливает
   принятое в корпус (+ финальный safety-gate детектором), затем `report.py` и
   `build_infographic.py`.
